    9 Easy steps to Setup EC2 instance

    In this article, we’re going to create an EC2 instance inside a subnet inside a VPC inside a availability zone inside a region inside AWS.

    AWS [ Region [ AZ [ VPC [ Subnet [ EC2 ] ] ] ] ]

    Basics

    Before we start, let’s understand some very minimum terminologies to understand some AWS jargon in easy language.

    • EC2: Elastic Compute Cloud. This is a service that provides virtual machines that you can use to run your applications.
      • AMI (Amazon Machine Image) is a template that contains the software configuration (for example, an operating system, an application server, and applications) for an EC2 instance. You can launch multiple instances from a single AMI.
      • An instance type is a type of EC2 instance that has a specific set of hardware specifications, such as the number of CPU cores, the amount of memory, and the storage capacity. Eg General Purpose, Compute/Memory/Storage/GPU Optimized. (Eg T2 micro, T2 small, etc.)
    • VPC: Virtual Private Cloud. This is a logically isolated section of the AWS Cloud where you can launch your EC2 instances.
    • Subnet: A logical division of a VPC. Subnets can be used to group EC2 instances together based on their function or security requirements. A range of IP address will be reserved for this subnet. An EC2 instance will be assigned to some IP address from this range.
    • Security group (SG): A set of rules that control inbound and outbound traffic for your EC2 instances. Security groups can be used to control who can access your EC2 instances and what resources they can access.
    • IAM roles: Permissions that can be assigned to EC2 instances. IAM roles allow you to grant permissions to EC2 instances without having to manually configure the permissions on each instance.
    • Key pair: A set of public and private keys that can be used to authenticate to your EC2 instances. Key pairs are used when you connect to your EC2 instances using SSH.
    • EBS: Elastic Block Store. This is a storage service that provides persistent block storage volumes for your EC2 instances. EBS volumes can be used to store data that needs to be kept persistent, even if your EC2 instance is terminated.
    • EFS: Elastic File System. This is a file storage service that provides scalable, highly available, and durable file storage for your EC2 instances. EFS can be used to share files between EC2 instances, and it can also be used to store data that needs to be accessed by multiple users or applications.

    Setup

    Prerequisite: I’m expecting that following are already setup;

    • AWS account
    • VPC
    • Subnet
    • IAM Role
    • Security Group

    Steps

    It is in summary what you’ve to do. Refer the quick reference image to visualize and understand it properly.

    1. Login to EC2 Dashboard: AWS account > AWS Services tab > Select “Compute” { list of service }
    2. Select AWS Region > Launch Instance
    3. Select AMI(Image): Set of OS, Software, libraries etc (just like VM image, Docker image, or ISO image)
    4. Choose Instance Type: Select the hardware capacity for your virtual machine.
    5. Configure instance details
      1. No. of instances: you can provision up to 20 instances at a time.
      2. Purchase Options: Request Spot Instances? Yes/No
      3. Networking details: In which VPC (Virtual Private Cloud) you want to launch your instance and under which subnets inside your VPC. AWS will assign a dynamic IP to it from their pool of IPs once the instance is launched.
        1. VPC,
        2. Subnet,
        3. Auto assign Pubic IP? Yes/No Auto assigned public IP can be changed when an instance restarts (shutdown -> starts). To assign a static IP check appendix.
      4. IAM Role: Access permissions
      5. Shutdown Behaviour: Stop or Terminate
        1. Enable termination protection: In case, you have accidentally terminated your instance, AWS has a layer of security mechanism. It will not delete your instance if you have enabled accidental termination protection.
      6. Under Monitoring: you can enable Detailed Monitoring if your instance is a business critical instance. Basic monitoring is free of cost.
      7. Under Tenancy: If your application is a highly secure application, then you should go for dedicated capacity otherwise shared tenancy.
    6. Add Storage: AWS provides 3 types of EBS volumes- Magnetic, General Purpose SSD, Provisioned IOPs.
    7. Tag Instance: To find an instance easily
    8. Create/Select Security Group (SG): Consider it as firewall to control incoming and outgoing connections/traffic. In this article, I consider that SG is already created.
    9. Add SSH keys: To access the virtual machine from terminal. Download the keys and add them to your login software or OS.

    To see all launched instances: AWS account > “Instances” from left panel

    ✔️ Tip

    Terminate the instances you don’t need to save cost.

    Appendix

    Static IP address

    This is an optional step

    Need: To assign a static IP address to a EC2 instance or VPC so that it is not changed when your virtual machine starts again.

    Steps

    1. Login to EIP Dashboard: AWS account > “Elastic IPs” from left panel
    2. Create a Static IP Address: Allocate New Address > VPC/EC2
    3. Bind address to an EC2: Select already created IP address > “Actions” menu > “Associate Address” > Search for your instance > “Associate”

    You can check the associated EIP from Instances dashboard.

    ✔️ Tip

    From the “Actions” menu of EIP dashboard, you can release or delete previous address to reduce cost.

    Login to EC2 instance

    You can check the status, and the configuration of EC2 instance from EC2 dashboard.

    1

    ssh -i /path/key-pair-name.pem instance-user-name@instance-public-dns-name
    • instance-public-dns-name: Use the IP address of your machine to connect.
    • You must have already downloaded the SSH keys during the instance creation. Setup them to the software or OS you’ll be using for login.
    • Open the port for SSH connection in security group. You can optionally add the IP address of the machine you’ll be accessing the EC2 instance from.
    • There is a default user name based on the AMI you used. Once you login with default user name, you can create more users for safety reasons.

    Default Users

    AMI user name
    Amazon Linux 2023, Amazon Linux 2, or the Amazon Linux AMI ec2-user
    CentOS AMI centos or ec2-user
    Debian AMI admin
    Fedora AMI fedora or ec2-user
    RHEL AMI ec2-user or root
    SUSE AMI ec2-user or root
    Ubuntu AMI ubuntu
    Oracle AMI ec2-user
    Bitnami AMI bitnami

    Otherwise, check with the AMI provider.

    ✔️ Tip

    Avoid using the default user & SSH keys if there are multiple users. For this, Create key pair > import the public key to EC2 instance > share private key with other users

    Snapshot

    Amazon provided AMIs are a standard set of software and OS which can’t be modified. Consider them like a docker image. We can create an instance from them just like we create docker container.

    What if we change anything in the instance and restart it?

    The changes will be lost. Every time, we restart an instance, it starts from a base image from scratch. To persist the changes, you can take the snapshot.

    You can either take the snapshot of complete EC2 instance which includes OS, applications and data or just the volume that include only data. If you take the snapshot of an instance, you can use it to create an AMI. You need not to stop the instance to take it’s snapshot. To take the snapshot of a volume, you must stop the running instance first.

    For a good practice, you should remove temp folder, cache, logs, or any temporary data before taking snapshot of an instance. So the size of the image is lesser.
